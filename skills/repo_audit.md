# Skill: Repo Audit

## Goal
Scan the repository and produce an engineering audit report.

## Steps
1. Identify entrypoints (scripts, cli, launch files).
2. Identify core modules and dependency structure.
3. List missing tests and missing docs.
4. Identify duplicated logic and refactor opportunities.
5. Provide a prioritized refactor plan.

## Output format
- Overview
- Architecture summary
- Critical issues (Top 5)
- Suggested refactor plan
- Quick wins (1 day)
- Long-term plan (1-2 weeks)
