# Generate Docs

Auto-generate documentation from code.

## Steps
1. Scan the project for source files, modules, and public APIs.
2. Analyze function signatures, classes, and module-level docstrings.
3. Generate or update documentation including:
   - Module-level overview
   - Function/method docstrings (Google or NumPy style)
   - Usage examples
   - API reference
4. Ensure consistency with existing documentation style.
5. Do NOT overwrite existing well-written docstrings.

## Output format
- Files documented
- Documentation style used
- Summary of additions/updates
- Any undocumented public APIs flagged
